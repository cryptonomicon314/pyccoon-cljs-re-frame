    Remember what event we are currently handling

    #

    Given an event vector, look up the handler, then call it. By default, handlers are not assumed to be pure. They are called with two paramters:

    • the app-db atom
    • the event vector

    The handler is assumed to side-effect on app-db - the return value is ignored. To write a pure handler, use the pure middleware when registering the handler.

    (defn handle

  [event-v]
  (let [event-id    (first-in-vector event-v)
        handler-fn  (lookup-handler event-id)]
    (if (nil? handler-fn)
      (error "re-frame: no event handler registered for: \"" event-id "\". Ignoring.")
      (if  *handling*
        (error "re-frame: while handling \""  *handling*  "\"  dispatch-sync was called for \"" event-v "\". You can't call dispatch-sync in an event handler.")
        (binding [*handling*  event-v]
          (handler-fn app-db event-v))))))
